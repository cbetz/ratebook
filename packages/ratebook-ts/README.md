# @ratebook/engine

The Ratebook rate engine, ported to TypeScript. Deterministic, exact-decimal, zero surprises —
the JS/edge/browser companion to the Python `ratebook` engine.

## The contract: the two engines never diverge

Both engines read the **same** golden JSON vectors
(`packages/ratebook/tests/vectors/v0_bills.json`, generated by the Python side) and must
reproduce every `expected` bill byte-for-byte. That's enforced here by `test/vectors.test.ts`.
The key to making it hold across languages: money and energy are `Decimal` (decimal.js), never
JS `number`, and serialize to the identical canonical string form Python emits.

```sh
pnpm install
pnpm test        # run the cross-engine vectors
pnpm build       # emit dist/ (+ .d.ts)
pnpm typecheck
```

Regenerate the shared vectors after any engine change:
`uv run python packages/ratebook/tests/generate_vectors.py`.

## Usage

```ts
import { Tariff, Usage, BillingWindow, estimateBill } from "@ratebook/engine";

const tariff = Tariff.fromJson(tariffJson);          // same JSON shape as the Python schema
const result = estimateBill(
  tariff,
  Usage.aggregate("900"),                            // or Usage.hourly([...]) for TOU plans
  BillingWindow.fromIso("2025-06-01", 30),
);
result.ok ? result.total!.toString() : result.refusal!.reason;  // "208.256" | a typed refusal
```

## Scope

v0 ports the schema, `estimateBill`, and `supported` — the bill-match core the vectors cover.
`estimateAnnual` (PySAM-parity mode) and the charge-window optimization will follow, each with
their own shared vectors so they're held to the same never-diverge bar.
